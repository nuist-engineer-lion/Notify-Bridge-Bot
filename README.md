# Notify-Bridge-Bot

一个基于 `napcat-sdk` 的 QQ 私聊客服催回复机器人。

它的目标很简单：客户发来私聊消息后，如果客服迟迟没有回复，机器人就把消息整理后推送到内部群里提醒；如果超时更久，再按里程碑继续催办；客服处理完成后，再自动结束监控。

## 适用场景

- QQ 私聊作为客服入口
- 需要把未回复客户集中提醒到内部群
- 希望减少人工盯聊天窗口的成本
- 希望用“贴表情”或“戳一戳”做快捷收尾

## 功能概览

- 自动通过好友申请，并同步通知到内部群
- 监听非白名单用户的私聊消息
- 对新消息做 1 分钟防抖，避免客户连续补充消息时频繁提醒
- 将待处理客户消息以合并转发形式发送到内部群
- 按里程碑进行超时催办
- 客服私聊回复后自动移出监控队列
- 支持在内部群对提醒消息贴表情，自动给客户发送结束语
- 支持在私聊窗口通过戳一戳，自动给客户发送结束语

## 工作流程

1. 客户发送私聊消息，机器人将其加入待回复队列。
2. 如果客户在 1 分钟内继续发消息，计时会重置。
3. 超过 1 分钟仍未回复时，机器人向内部群发送“新客户提醒”。
4. 超过配置的里程碑时间后，机器人继续发送“超时催办”。
5. 客服回复后，该客户会从监控队列中移除。
6. 客服也可以通过在内部群提醒消息上贴表情，或在私聊窗口戳一戳来结束会话。

## 运行要求

- Python `3.14+`
- 可用的 NapCat / OneBot WebSocket 服务
- 一个用于接收提醒的内部 QQ 群

## 安装

项目当前使用 `uv` 管理依赖。

```bash
uv sync
```

如果你已经有自己的 Python 环境，也可以手动安装依赖：

```bash
pip install napcat-sdk
```

## 配置

当前版本的配置写死在 [`main.py`](./main.py) 顶部，启动前至少需要检查这些常量：

- `WS_URL`：NapCat WebSocket 地址
- `WS_TOKEN`：WebSocket 鉴权 token
- `INTERNAL_GROUP_ID`：内部提醒群号
- `WHITELIST`：不参与监控的 QQ 号列表
- `MILESTONES`：超时催办里程碑，单位为分钟
- `MONITORED_FORWARD_LIMIT`：最多保留多少条可快捷处理的合并转发消息
- `CLOSING_MESSAGE`：快捷结束会话时发给客户的结束语

如果要部署到正式环境，建议至少做到两件事：

- 不要把真实 `WS_TOKEN` 直接提交到仓库
- 后续把这些配置改成环境变量或独立配置文件

## 启动

```bash
uv run python main.py
```

启动后程序会：

- 建立到 NapCat 的 WebSocket 连接
- 后台每 60 秒执行一次巡检
- 在连接断开后自动重连

## 提醒与收尾机制

### 新客户提醒

- 客户首次发消息后，不会立刻提醒
- 只有当“距离最后一条消息已满 1 分钟”时，才会推送到内部群
- 这样可以避免客户连续发多条消息时刷屏

### 超时催办

默认里程碑如下：

`5, 15, 30, 60, 120, 180, 360, 720, 1440, 2880, 4320` 分钟

每个客户每个里程碑只提醒一次；如果客户中途又发新消息，计时和里程碑记录都会重置。

### 会话结束

以下行为会让客户从待回复队列中移除：

- 客服直接私聊回复客户
- 通过内部群提醒消息触发自动结束语，并成功发出私聊
- 其他触发了机器人主动私聊发送消息的收尾动作

## 已知限制

- 当前状态全部保存在内存里，程序重启后待回复队列会丢失
- “贴表情自动结束”只支持单客户提醒消息；如果一条合并转发里包含多个客户，机器人会提示手动处理
- 配置目前硬编码在代码中，不适合多人协作或直接生产部署
